# SD-Menu

SD-Menu is Developer Tool for FiveM that allows you to support multiple menus and inputs using a single export and also adds [Font Awesome](https://fontawesome.com/) support to menu's without code changes.



## Supported Menus & Inputs

| Menu | Supported | Input | Supported |
| :-: | :-: | :-: | :-: |
| [nh-context v1]() | {GreenCheckIcon} | [nh-keyboard v1]() | {GreenCheckIcon} | 
| [nh-context v2](https://github.com/whooith/nh-context) | {GreenCheckIcon} | [nh-keyboard v2](https://github.com/whooith/nh-keyboard) | {GreenCheckIcon} | 
| [zf_context](https://github.com/zf-labo/zf_context) | {GreenCheckIcon} | [zf_dialog](https://github.com/zf-labo/zf_dialog) | {GreenCheckIcon} |
| [qb-menu](https://github.com/qbcore-framework/qb-menu) | {GreenCheckIcon} | [qb-input](https://github.com/qbcore-framework/qb-input) | {GreenCheckIcon} |
| [ox_lib](https://github.com/overextended/ox_lib) | {GreenCheckIcon} | [ox_lib](https://github.com/overextended/ox_lib) | {GreenCheckIcon} |

## Library (In Your Resource)
You can inlude sd-menu in your resource without having to depend on another resource, simply add **sd-menu.lua** to your resource and include it in your **fxmanifest.lua** on the client side.

```lua
client_scripts {
    'client/sd-menu.lua'
}
```

## Resource
You can also run sd-menu as standalone resource by downloading the github release

You can then use the exports as followed.

```lua
-- Menu
exports['sd-menu']:CreateMenu(data)

-- Input
exports['sd-menu']:CreateInput(data)
```
